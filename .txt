npm run build

# Build and fill netlify folder (for manual Netlify deploy)
npm run build:netlify

# Preview production build
npm run preview

# Companion (separate service)
npm run companion

---- COMPANION + LOCAL SERVER ----
# Flow: Controller sends state → (local) POST /api/companion-state → server writes to Supabase;
#       (Netlify) controller writes direct to Supabase. Railway Companion reads from Supabase (GET /state?code=...).
# Commands (Take/Next/Prev): Companion module → Railway HTTP → Railway broadcasts via Supabase Realtime → controller (local) receives.
# No Supabase "allowed origins" needed for local: the local server proxies companion state (dev 5173→3001 or build on 3000/3001).
# Optional: set VITE_COMPANION_API_URL in .env to your Railway URL, then rebuild; controller will also POST to Railway.
# Use the same connection code in the controller as in the Companion module config.
# For the LOCAL app to receive Companion commands (Take/Next/Prev): set VITE_COMPANION_API_URL in .env to your Railway URL, restart dev (or set localStorage.companionApiUrl and reload). The app will connect to Railway via WebSocket and receive commands (Realtime from localhost often doesn't deliver). Redeploy Companion to Railway after mount.js changes.
